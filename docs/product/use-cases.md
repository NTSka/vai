# Use Cases

This document captures initial user-facing use cases. Internal system processing
steps are documented as flows later and are not treated as user use cases unless
an actor directly initiates or observes them as part of a scenario.

## Create Organization

An organization owner creates or receives access to an organization in the
platform.

## Invite User

An organization owner invites another user to join the organization.

## Manage Organization Members

An organization owner changes member status or role assignments.

## Upload Document Set

An organization user uploads one or more files as a document set.

The uploaded set may contain individual documents, archives, or multi-document
files.

## View Uploaded Document Set Status

An organization user views whether a document set is uploaded, processing,
processed, or failed.

## View Documents in Project Structure

An organization user opens the project structure built from document codes and
views documents grouped under the relevant hierarchy branch.

## Find Related Documents Across Uploads

An organization user sees documents from different document sets grouped
together when their parsed codes point to the same project structure branch.

## Run Available Capability

An organization user runs or opens a configured capability when the required
documents and extracted data are available.

Initial example: compare working documentation and estimate documentation.

## Review Capability Result

A reviewer or organization user reviews a capability result, such as a detected
comparison mismatch, and accepts, rejects, or marks it for clarification.

## Export Capability Result

An organization user exports capability results into a report or external format.
